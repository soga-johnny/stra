## 3. 脆弱性

### 3-1. 参照系の恣意性（最大の脆弱性）

中央値は主観的に設定してよい、という柔軟性は強みである一方、  
**都合よく基準を置けば、どんな自己正当化も可能**になるリスクがある。

```
リスク例:
  - 負けているのに勝っている基準を選ぶ
  - 市場に通用していないのに内輪中央値で安心する
  - 組織が自己陶酔する
```

**現在の対策**: 外部フィードバックによる補正ループ（サイバネティック・ループ）を組み込むことで緩和。  
**残課題**: 補正を無視した場合の理論的歯止めが弱い。

---

### 3-2. 差分と価値の混同リスク

理論上は差分と価値を分けているが、実務では高確率で混線する。

```
誤認パターン:
  目立つ差分 = 価値ある差分（× 誤り）

実際は:
  差分があっても充足ニーズ（Job）と接続されない場合がある
  独自性があっても評価圏に入れない場合がある
  変であることと資本化されることは別
```

**対策の方向性**: 差分の「認識可能性 × 希少性 × 充足ニーズ（Job）との接続」の三条件チェックリスト化。

---

### 3-3. 時間概念の未精緻化

現時点で時間は「意味の耐久度指標」として位置づけられているが、  
以下の側面まで分解する必要がある。

| 側面 | 内容 | 現状 |
|---|---|---|
| 差分の**持続** | 時間が長いほどシグナル | ✅ 一部組み込み済み |
| 差分の**変容** | 差分は時間とともに内容が変わる | ❌ 未整備 |
| **参照系の変化** | 中央値自体が時間で変わる | ✅ 一部言及あり |
| 差分の**陳腐化** | 一度価値化された差分が平均化される | ❌ 未整備 |

**残課題**: 時間を単なる「耐久テスト」としてではなく、差分と基準の**双方を変質させる力**として精密化する。

---

### 3-4. 所属間の競合処理が未整理

複数所属は互いに矛盾することがある。

```
競合例:
  経営者としての合理性 vs クリエイターとしての美意識
  友人としての倫理   vs 市場プレイヤーとしての収益性
  長期思想家       vs 短期実務者
```

現時点では所属間の優先順位をどう決めるかのルールが未設計。  
**今後の方向性**: 「所属の重み（weight）× 文脈依存のダイナミックな優先順位」モデルの検討。

---

### 3-5. 評価圏の操作可能性（権力分析）

「評価圏が価値を決める」という立場は明快だが、  
評価圏それ自体が中立ではない点を看過していた。

```
評価圏を操作しうる権力構造:
  資本 / メディア / プラットフォーム / アルゴリズム / 制度 / ブルデュー的「場の支配者」
```

**v0.7.0での部分対処**: 象徴資本（`foundations/08-symbolic-capital.md`）の導入により、  
評価圏の非中立性のメカニズムが以下の構造として記述された。

- 象徴資本保有者は評価圏内の**目標差分水準の設定権**を持つ
- 同保有者は**設定差分の対応認定権**を持ち、どの差分の組み合わせが有効かを規定する
- これにより、所属中央値（統計的性質）を変えることなく、社会的評価の構造が非対称になる

**残課題**: 評価圏の形成・変容・象徴資本の蓄積を阻害する構造的条件の分析は未整備。  
象徴資本の循環を断ち切る設計原則の実践的適用基準も引き続き課題として残る。

---

### 3-6. 倫理レイヤー（v0.4.0で部分対処済み）

差分は善悪ではなく分類であり、価値は評価圏が決めるという構造層の立場は理論的にシャープである。  
ただし、社会設計に応用する際には倫理的な問いが生じる。

```
問題:
  扇動的差分（分断を生む差分）
  中毒性の高い差分（SNS・ゲームの過剰設計）
  他者破壊的な差分
  → これらも評価圏によっては資本化されうる
```

**v0.4.0での対処**: 設計原則（[foundations/00-principles.md](../foundations/00-principles.md)）において  
「豊かさ」を規範的目的として定義したことで、倫理レイヤーが原則レベルで設置された。

豊かさの5条件のうち「③選択肢の存在」「④意志の保持」「⑤実行基盤」は  
他者に対しても成立するべき条件であるため、  
**他者の選択肢を奪い・意志を損なう差分設計はStraの原則に反する**と明示された。

**残課題**: 原則は設定されたが、具体的な実践場面での判断基準・適用ルールの設計は未整備。  
「評価圏準拠型 vs 批評型」の分岐の運用設計が引き続き必要。

---

### 3-7. 模倣耐性の欠如

価値ある差分は模倣される。模倣されると差分は平均化される。  
現時点では差分の**消耗構造**が理論に組み込まれていない。

```
差分のライフサイクル（未実装）:
  差分の発生 → 評価圏への参入 → 資本化 → 模倣 → 平均化 → 新差分探索
```

**残課題**: このループを理論の動的構造として明示的に組み込む。

## 5. 理論発展の次のステップ

「**所属・時間・評価圏・権力**」の4軸で理論マップを構成することが  
次の体系化フェーズとして考えられる。

```
          評価圏
          ↑
所属 ←──────────→ 権力
          ↓
          時間
```

この4軸を整理することで、Straの応用領域（UX・組織・XR・社会設計）への  
具体的な接続経路が明確になる。
