## 1. 概念の位置づけ

### このドキュメントの目的

`practice/01-deviation-design.md` は「どんな入力差分を設計するか」を扱う（設計フェーズ）。  
本ドキュメントはその先、**設計した入力差分を時間単位で実行・観測・補正するサイクル**を扱う（実行フェーズ）。

理論的基盤は `foundations/07-execution-theory.md`（四諦・八正道のStra的解釈）に置く。本ドキュメントはその実行論を **1日の操作単位**に落とし込む。

```
practice/01-deviation-design.md  →  入力差分の「設計」
practice/02-task-execution.md    →  入力差分の「実行・観測・補正」
```

## 2. タスクシュートとStraの接続原理

タスクシュートは「今日実行可能なタスクのみを計画し、見積もり時間と実績時間を記録することでリアルタイムに実行計画を補正する」という設計思想を持つ。この思想はStraの差分論と以下の点で構造的に合致する。

| タスクシュートの要素 | Straにおける対応 |
|---|---|
| 時間の有限性（1日 = 固定リソース） | 有限性（人間固有の差分源泉）・実行基盤（条件⑤）の時間的表れ |
| タスク | 入力差分の最小実行単位 |
| 見積もり時間 | 入力差分の設計期待値（計画された差分投入量） |
| 実績時間 | 出力差分の観測値（実際に投入した差分量） |
| 見積もり − 実績 の差分 | 実行精度フィードバック。サイバネティックループの最小単位 |
| 完了ログの蓄積 | 差分統合量の記録 |
| 未達タスクの繰越し | 差分ギャップの持続（苦諦）の可視化 |

> 「今日できることだけを計画する」というタスクシュートの原則は、  
> 豊かさ条件⑤（実行基盤）の有限性を直視した設計思想として機能する。

## 3. タスクの記述方法

Straにおいてタスクは「入力差分の最小実行単位」として記述する。

### タスクの構成要素

| 要素 | 内容 |
|---|---|
| **対象差分ギャップ** | このタスクがどの差分ギャップ（苦諦）を縮小するか |
| **評価圏** | どの評価圏への資本変換を意図しているか |
| **行動内容** | 実際に行う行動の記述（正業の実践単位） |
| **見積もり時間** | 設計された入力差分量（時間） |
| **実績時間** | 観測された出力差分量（時間）。実行後に記録する |

### タスク記述の例

| 一般的な記述 | Stra的な記述 |
|---|---|
| 英語の勉強 30分 | 職業市場評価圏・言語差分ギャップ縮小 / 見積30分 |
| 提案書を書く 90分 | クライアント評価圏・価値提示差分の発信（正語） / 見積90分 |
| SNS投稿 20分 | 認知評価圏・差分の表出（正語） / 見積20分 |
| 専門書を読む 60分 | 学術・職業評価圏・文化資本差分の蓄積 / 見積60分 |

> Stra的な記述は**タスクを差分ギャップとの接続で定義する**ことで、  
> 「なぜこのタスクをするか」が常に可視化される。漫然とした行動リストを防ぐ効果を持つ。

## 4. 差分実行サイクル（1日の構造）

```
【朝：設計フェーズ】 正見 + 正思惟
  今日の入力差分リストを設計する
        ↓
【日中：実行フェーズ】 正業 + 正念 + 正語
  逐次実行 → 実績時間の即時記録
        ↓
【夜：統合フェーズ】 正見 + 正精進
  差分（見積もり − 実績）を観察し翌日の設計に反映する
```

---

### 朝：設計フェーズ

**対応する八正道の軸**：正見（現状観測）、正思惟（意図設定）

**目的**：今日実行可能な入力差分リストを設計する。

**手順**：

#### ① 利用可能時間を確認する（実行基盤の確認）

- 固定イベント・移動・食事・睡眠を除いた可処分時間を算出する
- 「今日実行可能なタスクのみを計画する」（タスクシュートの原則）
- 可処分時間を超える計画は、正命（実行基盤の維持）に反する

#### ② 差分ギャップのリストを確認する（正見の実践）

- 昨日の未達タスク・現在の差分ギャップリストを確認する
- `practice/01-deviation-design.md` で設計した入力差分の中から今日実行する分を選択する
- 各タスクが「どの苦諦（差分ギャップ）」に対応するかを確認する

#### ③ 今日のタスクと見積もり時間を設計する（正思惟の実践）

- タスクをリストアップし、各タスクに見積もり時間を設定する
- 見積もり時間の合計が①の可処分時間を超えないことを確認する
- 合計が超過する場合は優先度の低いタスクを翌日に繰り越す

---

### 日中：実行フェーズ

**対応する八正道の軸**：正業（実行）、正念（気づき）、正語（発信）

**目的**：タスクを逐次実行し、実績時間を即時記録する。

**手順**：

#### ① タスクを逐次実行する（正業の実践）

- タスクは原則として1つずつ順番に実行する
- 並行作業は実績時間の記録精度を下げ、差分ギャップとの接続意識を弱める
- タスクに着手したら開始時刻を記録する

#### ② 実績時間を即時記録する（正念の実践）

- タスク完了時に実績時間を記録する
- 実行中、「このタスクはどの差分ギャップに対応しているか」を意識する（正念）
- 中断・割り込みが発生した場合も、その時間を記録する

#### ③ リアルタイム補正を行う（サイバネティックループ）

- 実績が見積もりを超過した場合、残タスクを今日中に実行可能かを即座に判断する
- 不可能な場合は翌日に繰り越す（差分ギャップの持続として記録する）
- 見積もりの大幅な超過が続く場合は、入力差分の設計自体の見直しを検討する

---

### 夜：統合フェーズ

**対応する八正道の軸**：正見（観測）、正精進（統合・継続）

**目的**：実行の差分を観察し、差分統合量として積み上げ、翌日の設計に反映する。

**手順**：

#### ① 見積もり vs 実績の差分を観察する（正見の実践）

- 各タスクの `見積もり時間 − 実績時間` を確認する
- 差分が大きいタスクの傾向を把握する

| 傾向 | 示唆される補正 |
|---|---|
| 特定カテゴリのタスクの超過が続く | そのカテゴリの見積もり精度の更新が必要 |
| 全体的に超過が続く | 1日の計画量が実行基盤（可処分時間）を超過している |
| 完了タスクが少ない | 入力差分の粒度が大きすぎる（タスクを細分化する） |
| ギャップがほぼゼロ | 見積もり精度が高い状態。プラズミズムの向上サイン |

#### ② 差分統合量を記録する（正精進の実践）

- 今日の完了タスク数・投入時間・対象差分ギャップへの貢献を記録する
- プラズミズムの状態（今日の自分の適応状態の推定）を簡単に記録する
- 蓄積ログは差分統合量の記録として機能する

#### ③ 翌日の設計に反映する（補正ループの完成）

- 未達タスクの優先度を判断し、翌日のリストに追加する
- 繰り返し超過するタスクは「入力差分の設計見直し」の対象とする
- 中断・割り込みが多い環境であれば、正命（実行基盤設計）の見直しを検討する

## 5. プラズミズムとの接続

現在の適応状態（プラズミズムの4状態）によって、差分実行サイクルの実装方法を調整する。

| 適応状態 | 差分実行サイクルの特徴と推奨 |
|---|---|
| **固体** | タスク設計そのものが難しい。単一の評価圏への接触タスクを1〜2個から始める。見積もり時間は長めに設定する |
| **液体** | タスクは設計できるが見積もり精度が低い。15〜30分の短時間タスクを中心に実績データを蓄積する |
| **気体** | 複数評価圏のタスクを並行設計できる。見積もり精度が向上し始める。八正道の全軸を意識した実行が可能になる |
| **プラズマ** | 設計→実行→補正のサイクルを自律的に回せる。見積もりと実績の差分が最小化される状態 |

> プラズミズムの適応状態は、差分実行サイクルの**実行精度（見積もり vs 実績の差分）**によって観測できる。  
> 差分が縮小傾向にあれば適応状態が向上していると推定できる。

## 6. 八正道の日次サイクルへの落とし込み

`foundations/07-execution-theory.md` で定義した八正道の各軸は、1日のサイクルの中で以下のように機能する。

| 正道軸 | 日次サイクルでの実践 |
|---|---|
| **正見** | 朝：差分ギャップの現状確認 / 夜：見積もり−実績の観察 |
| **正思惟** | 朝：今日の入力差分の設計意図の設定 |
| **正語** | 日中：設計した差分を評価圏に発信するタスクの実行 |
| **正業** | 日中：タスクの逐次実行 |
| **正命** | 全体：可処分時間の確保・睡眠・実行基盤の維持 |
| **正精進** | 夜：差分統合量の記録・翌日への継続コミット |
| **正念** | 日中：実行中のタスクの差分的意味への継続的注意 |
| **正定** | 日中：単一タスクへの集中状態の維持 |

---

## 注記

差分実行サイクルは「入力差分を時間単位で設計・実行・補正する枠組み」を提供する。  
実行の継続力・動機・感情の変動は、プラズミズムの適応状態（`foundations/06-plasmism.md`）として扱い、  
実行サイクルの「見積もり vs 実績差分」が適応状態の観測指標として機能する。

また、差分実行サイクルは豊かさの5条件を整えるための手段である。  
他者の実行基盤を奪い・実行を妨害する設計はStraの原則に反する（`foundations/00-principles.md § 4`）。
