## 1. 概念の位置づけ

`sphere/` は差分構造が社会・評価圏・資本をいかに形成しているかの客観的記述体系を扱う。  
`locus/` は差体が豊かさを追求するための方法論体系を扱う。

`currency/` はその**循環層**として機能する。差体が入力差分を投じ（実行）、評価圏が出力差分を認識し（資本変換）、新たな評価圏へアクセスする（展開）という一連の循環を、**単一の追跡可能な流量単位**として記述するのが通貨の役割である。

```
sphere/   →  [ 圏観: 評価圏・変換構造・象徴資本 ]
locus/    →  [ 差体: 実行・適応・日次サイクル  ]
              ↕ 接続・循環
currency/ →  [ 流量: 入出力の追跡・変換・蓄積・使用 ]
```

---

## 2. 定義

> **通貨**: 差体が投じた入力差分と評価圏から回収した出力差分の流量を追跡する主観的単位。  
> 差体の実行エネルギーの循環を一元的に記述するために導入される。


---

## 3. 通貨の本質的性質

1. **主観的追跡単位** — 差体ごとに独立した通貨系。他の差体との共通通貨や交換を前提としない
2. **差分の流量代理変数** — 通貨は資本そのものではなく、資本変換の流量過程を追跡する代理変数
3. **最大化の奨励** — 差体は通貨の最大化を目指してよい。ただし通貨 ≠ 豊かさ（後述）
4. **評価圏依存** — 通貨の生成・消費レートは所属と評価圏の組み合わせによって変動する
5. **両替可能** — 所属変換定義と評価圏変換定義を介した換算が可能。固定レートではなく変動する

---

## 4. 複式モデル

通貨の流向は**複式簿記的**に記述される。

```
借方（消費・支出）                 貸方（収益・回収）
─────────────────────────────────────────────────────
入力差分の実行       ←→           出力差分の評価圏認識
（通貨の消費）                     （通貨の生成）

評価圏アクセスコスト ←→           新評価圏での変換機会
（通貨の消費）                     （将来の通貨生成源）
```

| イベント | 通貨の動き | 説明 |
|---|---|---|
| 入力差分の実行 | 消費（支出） | 時間・認知・エネルギーの投入 |
| 出力差分が評価圏に認識される | 生成（収入） | 変換レート × 差分量 |
| 差分ギャップの持続 | 未回収ポジション | 投資中・未精算のポジション |
| 評価圏アクセス | 消費（支出） | 新評価圏への参入コスト |
| 差分統合量への累積 | 通貨履歴の蓄積 | プラズミズム状態遷移への間接的貢献 |

### 差分ギャップと未回収ポジション

```
入力差分の消費     →  ポジションが開く（未回収状態）
        ↓
評価圏が認識       →  ポジションが閉じる（回収完了）
        ↓
回収通貨 > 消費通貨  →  差分収益（プラス収支）
回収通貨 < 消費通貨  →  差分損失（ポジション見直しのサイン）
```

ポジションが長期間閉じない場合は、評価圏の選択・目標差分水準の観測・入力差分の設計のいずれかに問題がある可能性が高い。

## 5. 通貨と資本・豊かさの関係

### 通貨 ≠ 資本

資本（経済・文化・社会関係）は評価圏において実態として生成されるもの。  
通貨はその変換過程の**流量を追跡する記録単位**であり、資本そのものではない。

```
差体の実行   →  通貨：消費
      ↓
出力差分     →  通貨：回収
      ↓
資本変換     ←  ここで初めて資本（経済・文化・社会関係）が現れる
      ↓
資本構成の変化（通貨はこの過程の流量を記録するだけ）
```

### 通貨 ≠ 豊かさ

豊かさは5条件の同時充足という構造的状態であり、通貨残高の大小ではない。

ただし通貨の最大化を追求することは、評価圏との接触深度を高め、差分統合量を蓄積し、豊かさの複数条件の充足に間接的に寄与する設計になっている。

| 豊かさの条件 | 通貨との関係 |
|---|---|
| ① 所属の認識 | 通貨のポジション管理が所属の認識精度を高める |
| ② 認識の更新可能性 | 未回収ポジションのフィードバックが推定補正を促す |
| ③ 選択肢の存在 | 評価圏アクセスコストの蓄積が選択肢を拡張する |
| ④ 意志の保持 | 通貨量は意志を保証しない |
| ⑤ 実行基盤 | 通貨の複式管理が実行基盤の定量的把握を支援する |
